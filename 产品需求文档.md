## 1. 产品概述

### 1.1 产品名称

**InspireHub**（灵感中心）

### 1.2 产品口号

**沉淀灵感，连接设计与开发。**

### 1.3 产品定位

InspireHub 是一个专为设计师和开发者打造的**通用灵感收集与管理工具**。用户可以通过浏览器快速收藏网页、图片、代码片段、文本笔记，并通过标签、领域（设计/开发/产品）和全文搜索进行高效检索。它既是个人知识库，也是创意启动器。

### 1.4 目标用户

- **UI/UX 设计师**：收藏优秀界面设计、配色方案、图标资源、设计文章。
- **前端/后端开发者**：保存代码片段、命令行脚本、技术文档链接、开源项目。
- **产品经理/创作者**：收集产品案例、数据报告、金句、思维模型。

### 1.5 核心价值

- **极速收集**：支持手动添加、浏览器扩展（后期）一键剪藏。
- **多维组织**：类型 + 领域 + 自定义标签，灵活分类。
- **高效检索**：全文搜索 + 多维度筛选，找回灵感快如闪电。
- **纯净体验**：无广告、无社交干扰，专注内容本身。
- **数据归属**：支持导出备份，可选云同步（自托管或 SaaS）。

## 3. 功能需求

### 3.1 灵感管理（核心）

#### F-01 添加灵感

- **入口**：顶部导航栏的“+ 添加灵感”按钮，全局悬浮按钮。
- **表单字段**：
  - 标题（必填，文本，最长 200 字符）
  - 类型（必填，单选：链接 / 图片 / 代码 / 文本笔记）
  - 内容（必填，多行文本框）：
    - 链接类型：URL 输入框
    - 图片类型：图片 URL 或 本地上传
    - 代码类型：代码编辑器（支持语法高亮）
    - 文本笔记：富文本 / Markdown 编辑器
  - 领域（多选：设计 / 开发 / 产品）
  - 标签（可选，文本输入，支持自动补全已有标签）
  - 备注（可选，Markdown）
  - 来源 URL（可选，记录原始引用）
- **逻辑**：
  - 如果类型为“图片”且用户上传本地图片，后端压缩并存储，生成访问 URL。
  - 如果类型为“链接”，后端尝试抓取标题和摘要（可做可不做 MVP）。
- **验收**：表单提交后，数据保存至数据库，并在列表中显示新卡片。

#### F-02 灵感列表展示

- **视图**：
  - 卡片视图（默认）：每个卡片显示标题、类型图标、领域标签、内容预览（图片显示缩略图，代码显示前两行）。
  - 列表视图（简洁）：仅显示标题、类型、创建时间。
- **分页**：每页 20 条，支持滚动加载更多（无限滚动）或传统分页。
- **排序**：按创建时间倒序（默认）、按更新时间、按标题字母序。

#### F-03 筛选与搜索

- **快速筛选**：
  - 领域筛选：设计 | 开发 | 产品（点击切换，可多选）。
  - 类型筛选：链接 | 图片 | 代码 | 笔记（多选）。
- **标签筛选**：
  - 侧边栏展示**标签云**（按使用频率排序），点击标签即可筛选。
  - 支持多标签组合筛选（AND 逻辑）。
- **全文搜索**：
  - 搜索框输入关键词，检索范围：标题 + 备注 + 代码内容 + 文本笔记内容。
  - 支持模糊匹配，高亮关键词（前端实现）。

#### F-04 灵感详情

- 点击卡片进入详情页（路由：`/inspirations/:id`）。
- 根据类型展示不同内容：
  - 链接：显示目标网页标题、描述，可点击跳转，并支持内嵌预览（iframe）。
  - 图片：大图显示，显示图片尺寸（后端存储时可记录）。
  - 代码：语法高亮 + 行号 + 一键复制按钮。
  - 文本笔记：Markdown 渲染。
- 展示完整信息：创建时间、更新时间、标签、领域、备注。
- 操作按钮：编辑、删除、复制链接、导出为 Markdown。

#### F-05 编辑与删除

- 编辑：表单预填充现有数据，支持修改任意字段，更新 `updated_at`。
- 删除：弹出二次确认，软删除或物理删除（MVP 可物理删除）。

#### F-06 数据导入/导出

- **导出**：支持导出所有灵感为 JSON 文件，或按筛选结果导出。
- **导入**：支持从 JSON 文件批量导入（合并或覆盖选项）。

### 3.2 用户与认证

#### F-07 注册与登录

- **注册**：用户名、邮箱、密码（长度 6-20 位），邮箱唯一。
- **登录**：邮箱+密码，登录成功返回 JWT token。
- **保持登录**：token 有效期 7 天，前端自动刷新（可选）。
- **登出**：前端清除 token，后端无需操作。

#### F-08 用户个人设置

- 修改昵称、头像（后期）、修改密码。
- 删除账号（软删除所有关联灵感）。

### 3.3 扩展功能（后续版本）

#### F-10 每日随机灵感

## 4. 非功能需求

### 4.1 性能

- API 响应时间：列表查询 < 500ms（1000 条数据内），搜索 < 800ms。
- 支持 100 并发用户。
- 图片上传大小限制：单张 ≤ 10MB。

### 4.2 可用性

- 页面加载完成时间 < 2 秒（首屏）。
- 支持移动端响应式（手机浏览器可正常使用核心功能）。

### 4.3 安全性

- 用户密码使用 BCrypt 加密存储。
- JWT token 携带用户 ID，所有 API 需验证 token 有效性。
- 用户只能访问自己的灵感数据（Service 层强制过滤 `user_id`）。
- 上传文件需校验类型（仅图片），防止恶意文件。

### 4.4 可扩展性

- 数据库设计预留字段（如 `meta` JSON 列）。
- 代码层次清晰，便于后续增加新类型（如“音频”）。

## 6. 数据模型（核心字段）

### 6.1 用户表 (users)

| 字段          | 类型         | 说明           |
| :------------ | :----------- | :------------- |
| id            | BIGSERIAL    | PK             |
| username      | VARCHAR(50)  | 唯一           |
| email         | VARCHAR(100) | 唯一，登录凭证 |
| password_hash | VARCHAR(255) | BCrypt 哈希    |
| avatar_url    | VARCHAR(500) | 可选           |
| created_at    | TIMESTAMP    |                |

### 6.2 灵感表 (inspirations)

| 字段            | 类型         | 说明                          |
| :-------------- | :----------- | :---------------------------- |
| id              | BIGSERIAL    | PK                            |
| user_id         | BIGINT       | FK → users.id                 |
| title           | VARCHAR(200) |                               |
| type            | VARCHAR(20)  | link / image / code / note    |
| content         | TEXT         | URL / 图片URL / 代码 / 纯文本 |
| domain          | TEXT[]       | 例如 `{'design','dev'}`       |
| tags            | TEXT[]       | 例如 `{'tailwind','color'}`   |

| language        | VARCHAR(30)  | code 类型时记录语言           |
| source_url      | VARCHAR(500) | 原始引用链接                  |
| image_thumbnail | VARCHAR(500) | 自动生成的缩略图 URL          |
| created_at      | TIMESTAMP    |                               |
| updated_at      | TIMESTAMP    |                               |

> 说明：数组类型利用 PostgreSQL 原生支持，便于索引和查询。

## 7. REST API 设计（主要端点）

| 方法   | 路径                                 | 描述                            | 认证 |
| :----- | :----------------------------------- | :------------------------------ | :--- |
| POST   | `/api/auth/register`                 | 用户注册                        | 否   |
| POST   | `/api/auth/login`                    | 登录，返回 token                | 否   |
| GET    | `/api/user/profile`                  | 获取当前用户信息                | 是   |
| PUT    | `/api/user/profile`                  | 更新用户信息                    | 是   |
| GET    | `/api/inspirations`                  | 分页获取灵感列表（支持筛选）    | 是   |
| GET    | `/api/inspirations/{id}`             | 获取单条灵感                    | 是   |
| POST   | `/api/inspirations`                  | 创建灵感                        | 是   |
| PUT    | `/api/inspirations/{id}`             | 更新灵感                        | 是   |
| DELETE | `/api/inspirations/{id}`             | 删除灵感                        | 是   |
| GET    | `/api/inspirations/search?q=keyword` | 全文搜索                        | 是   |
| POST   | `/api/upload/image`                  | 上传图片（multipart/form-data） | 是   |
| GET    | `/api/tags`                          | 获取当前用户所有标签（含频次）  | 是   |
| GET    | `/api/export`                        | 导出所有灵感为 JSON             | 是   |
| POST   | `/api/import`                        | 导入 JSON 文件（合并）          | 是   |

**筛选参数示例**：
`GET /api/inspirations?type=image&domain=design&tags=color,gradient&page=1&size=20`

**响应统一结构**：

json

```
{
  "code": 200,
  "message": "success",
  "data": { ... }
}
```

## 8. UI/UX 设计指引

### 8.1 整体风格

- 深色主题（默认），浅色主题可选。
- 背景色 `#1A1B26`，卡片 `#2A2B36`，主色调 `#7AA2F7`。
- 字体：系统默认无衬线（Inter），代码区域 `JetBrains Mono`。

### 8.2 页面布局

- **顶部栏**：Logo（左侧）、搜索框（中央）、添加灵感按钮 + 用户头像（右侧）。
- **左侧边栏**（可折叠）：快速领域切换、类型筛选、标签云。
- **主区域**：灵感卡片网格（响应式，最小 280px 宽度）。
- **右下角悬浮**：添加快捷按钮（“+”）。

### 8.3 关键交互

- 添加灵感：侧滑抽屉（Drawer）或模态框，避免页面跳转。
- 卡片悬停：轻微上浮 + 阴影，显示“编辑”图标。
- 删除：二次确认 toast + 撤销功能（5 秒内可撤销）。
- 搜索：输入时防抖（300ms），实时显示结果。

## 10. 验收标准

- 用户可以注册/登录，且不同用户的数据相互隔离。
- 用户可以添加四种类型的灵感，并能上传本地图片。
- 灵感列表正确展示，支持分页。
- 按领域、类型、标签筛选功能正常，多条件组合正确。
- 全文搜索能命中标题/备注/代码内容。
- 用户能编辑、删除自己的灵感。
- 导出为 JSON，导入后数据完整。
- 所有 API 均有 JWT 保护，未授权访问返回 401。
- 前端页面在 Chrome、Firefox、Safari 最新版无明显样式错乱。
- 手机端（宽度 375px）可正常浏览和添加灵感。

## 11. 风险与应对

| 风险                             | 概率 | 影响 | 应对措施                                                    |
| :------------------------------- | :--- | :--- | :---------------------------------------------------------- |
| 全文搜索性能下降（数据量 > 10w） | 低   | 中   | MVP 阶段限制用户最多 5000 条；后期迁移至 Elasticsearch      |
| 图片存储占用服务器空间           | 中   | 中   | 限制单张 5MB，自动压缩；生产环境使用对象存储（OSS）         |
| JWT 被盗用                       | 低   | 高   | 使用短期 token（7 天）+ refresh token（后期），支持强制登出 |
| 前后端联调时间超出预期           | 中   | 中   | 提供 Swagger 文档，Mock 前端开发                            |
