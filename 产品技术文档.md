# InspireHub 技术栈文档（Spring Boot 4 版）

> 版本：V3.0
> 更新日期：2026-06-08
> 适用范围：灵感板项目（设计与开发领域素材收集）
> 说明：本版从 Spring Boot 3.x 升级至 Spring Boot 4.0.x，同步更新所有依赖及 Spring Security 7 破坏性变更。

------

## 1. 总体架构

前后端分离架构，持久化、缓存、存储、部署采用成熟方案。

text

```
[浏览器/PWA] → (HTTPS + JSON) → [Nginx(可选)] → [Spring Boot 4 后端] → [Redis] → [PostgreSQL]
                                         ↓
                                   [对象存储 OSS]
```



- **前端**：Vite + React SPA，支持 PWA 离线访问。
- **后端**：Spring Boot 4 + MyBatis-Plus（spring-boot4-starter），集成 Redis 缓存、OSS 对象存储。
- **通信**：RESTful API + JWT 认证。
- **部署**：容器化（Docker Compose），CI/CD 自动化。

## 3. 后端技术栈（Spring Boot 4 升级版）

### 3.1 Spring Boot 4 重大变更速览

> ⚠️ 从 Spring Boot 3.x 升级至 4.0.x 的主要破坏性变更：

| 变更项 | Spring Boot 3.x | Spring Boot 4.x |
| :--- | :--- | :--- |
| 基础框架 | Spring Boot 3.2+ | **Spring Boot 4.0.x**（当前 4.0.6） |
| Spring Framework | 6.2.x | **7.0.x** |
| Jakarta EE | 9/10（Servlet 5/6） | **11（Servlet 6.1）** |
| Spring Security | 6.2+ | **7.0.x** |
| Tomcat | 10.1.x | **11.0.x** |
| Undertow | 支持 | **已移除**（不兼容 Servlet 6.1） |
| Hibernate | 6.6.x | **7.1.x / 7.2.x** |
| Jackson | 2.x（com.fasterxml） | **3.x**（tools.jackson），Jackson 2 已弃用 |
| MyBatis-Plus Starter | `mybatis-plus-spring-boot-starter` | **`mybatis-plus-spring-boot4-starter`** |
| Security DSL | `and()` + `authorizeRequests` 可用 | **仅 Lambda DSL**，`and()`/`authorizeRequests` 已移除 |
| 请求匹配器 | `MvcRequestMatcher` / `AntPathRequestMatcher` | **仅 `PathPatternRequestMatcher`** |
| 测试 Mock | `@MockBean` / `@SpyBean` | **已移除**（使用 `@MockitoBean` / `@MockitoSpyBean` 或手动注入） |
| 嵌入式启动脚本 | 支持 | **已移除**（使用 `java -jar`） |

### 3.5 安全与认证（Spring Security 7 重大变更）

| 类别     | 技术选型                       | 说明 |
| :------- | :----------------------------- | :--- |
| 安全框架 | **Spring Security 7.0.x**      | ⚠️ 仅 Lambda DSL，`and()` 已移除 |
| JWT 工具 | JJWT 0.12.3                    | API 稳定，无破坏性变更 |
| 密码编码 | BCryptPasswordEncoder          | 无变更 |
| SPA CSRF | `http.csrf(csrf -> csrf.spa())` | SB 4 新增，专为 SPA 设计 |

**Spring Security 7 配置示例（Lambda DSL 强制）**：

```java
@Bean
public SecurityFilterChain filterChain(HttpSecurity http) throws Exception {
    http
        .csrf(csrf -> csrf.spa())                          // SB 4 新增 SPA CSRF
        .authorizeHttpRequests(auth -> auth
            .requestMatchers("/api/auth/**").permitAll()
            .anyRequest().authenticated()
        )
        .sessionManagement(session -> session
            .sessionCreationPolicy(SessionCreationPolicy.STATELESS)
        )
        .addFilterBefore(jwtAuthenticationFilter, UsernamePasswordAuthenticationFilter.class);
    return http.build();
}
```

> ⚠️ **关键注意**：Spring Security 7 已移除 `and()` 链式调用和 `authorizeRequests`，必须使用上述 Lambda 风格。`AntPathRequestMatcher` 也已移除，统一使用 `PathPatternRequestMatcher`。

## 5. 部署与运维（升级版）

### 5.1 容器化（推荐）

提供 `Dockerfile` 和 `docker-compose.yml`：

yaml

```
services:
  postgres:
    image: postgres:14
    environment:
      POSTGRES_DB: inspirehub
      POSTGRES_USER: inspire_user
      POSTGRES_PASSWORD: ${DB_PASSWORD}
    volumes:
      - pgdata:/var/lib/postgresql/data

  redis:
    image: redis:7-alpine
    ports:
      - "6379:6379"

  backend:
    build: ./backend
    depends_on:
      - postgres
      - redis
    environment:
      SPRING_DATASOURCE_URL: jdbc:postgresql://postgres:5432/inspirehub
      SPRING_REDIS_HOST: redis
    ports:
      - "8080:8080"

  frontend:
    build: ./frontend
    ports:
      - "80:80"
    depends_on:
      - backend

volumes:
  pgdata:
```

> 注：Docker Compose V2 不再需要 `version` 字段。

### 5.2 CI/CD 流水线（GitHub Actions 示例）

yaml

```
name: CI/CD

on:
  push:
    branches: [ main ]

jobs:
  test:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - name: Set up JDK 25
        uses: actions/setup-java@v4
        with:
          java-version: 25
      - name: Maven test
        run: mvn test
      - name: Node.js test
        run: npm test

  build-and-deploy:
    needs: test
    steps:
      - name: Build backend jar
        run: mvn clean package
      - name: Build frontend
        run: npm run build
      - name: Deploy to server via SSH
        uses: appleboy/scp-action@v0.1.5
        with:
          source: "target/*.jar,dist/*"
          target: "/app"
```

> 注：CI 使用 JDK 25（Spring Boot 4 首等支持 Java 25）。

## 6. 技术决策与升级理由（更新）

| 技术点                  | 引入阶段         | 理由                                              |
| :---------------------- | :--------------- | :------------------------------------------------ |
| **Spring Boot 4**       | 项目启动时       | 最新主版本，Spring Framework 7 + Jakarta EE 11 底座 |
| **Spring Security 7**   | 项目启动时       | Lambda DSL 更简洁，SPA CSRF 原生支持               |
| **Jackson 3**           | 项目启动时       | SB 4 默认，Jackson 2 已弃用                        |
| shadcn/ui               | MVP 开发         | 快速构建一致、美观的 UI，减少自定义样式工作量      |
| Redis 缓存              | MVP 后立即       | 标签云频繁查询，缓存可降低数据库压力               |
| MyBatis-Plus 代码生成器 | 项目启动时       | 避免手写基础 CRUD，提高效率                        |
| 对象存储（OSS）         | 上线前           | 本地存储无法扩展，生产必须切换                     |
| CI/CD                   | 上线前           | 自动化测试与部署，保证发布质量                     |
| PWA                     | 迭代阶段         | 提升用户粘性，支持离线访问                         |
| 虚拟滚动                | 用户数据 >500 时 | 保证大列表滚动流畅                                 |

## 9. Spring Boot 3 → 4 迁移备忘

| # | 迁移项 | 操作 |
| :--- | :--- | :--- |
| 1 | Starter 重审 | SB 4 模块化拆分，检查所有 starter（如 Flyway/Liquibase 需显式引入） |
| 2 | MyBatis-Plus starter | 替换为 `mybatis-plus-spring-boot4-starter`（3.5.15+） |
| 3 | Spring Security 配置 | 移除 `and()` / `authorizeRequests` / `AntPathRequestMatcher`，改为 Lambda DSL |
| 4 | Jackson 2 → 3 | `com.fasterxml` → `tools.jackson`，`ObjectMapper` → `JsonMapper.Builder` |
| 5 | `@MockBean` / `@SpyBean` | 替换为 `@MockitoBean` / `@MockitoSpyBean` 或 `@TestConfiguration` |
| 6 | Undertow → Tomcat | 移除 Undertow 依赖，使用默认 Tomcat 11 |
| 7 | `application.properties` | 添加 `spring-boot-properties-migrator` 检测重命名属性 |
| 8 | 嵌入式启动脚本 | 改用 `java -jar` 或 Gradle application plugin |
